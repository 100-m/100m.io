Buttons

Button sizes

Define a button size with .button--xs , .button--s , .button--m , .button--l or .button--xl . The default button size is .button--m .

Extra Large Button

Large Button

Medium Button

Small Button

<button class="button button--xl button--flat bg-marvel">...</button>
<button class="button button--l button--flat bg-marvel">...</button>
<button class="button button--m button--flat bg-marvel">...</button>
<button class="button button--s button--flat bg-marvel">...</button>

Flat button

Flat buttons should be used to signify necessary or mandatory actions. There are two color options, specified with an additional class .bg--marvel or .bg--positive .

Default
Positive 
<button class="button button--m button--flat bg-marvel">...</button>
<button class="button button--m button--flat bg-positive">...</button>

Ghost button

Ghost buttons should be used to signify optional, infrequent or subtle actions. The color options are specified with an additional class .bg--marvel--hover , .bg--orange--hover , .bg--negative--hover , etc.

Default
Orange
Positive
Negative 
<button class="button button--m button--ghost bg-marvel--hover" type="button">...</button>

Ghost button on dark background

This is the reversed version of ghost buttons which is meant to be used on dark backgrounds or reversed themes. The color options are specified with an additional class .bg--marvel--hover , .bg--orange--hover , .bg--negative--hover , etc.

Default 
<button class="button button--m button--ghost bg-white--hover is-reversed" type="button">...</button>

Outlined button

Outlined buttons are very similar to ghost buttons and are used on light grey backgrounds where a button background has to be applied. The color options are specified with an additional class .bg--marvel--hover , .bg--orange--hover , .bg--negative--hover , etc.

Default
Orange
Positive
Negative 
<button class="button button--m button--outlined bg-white bg-marvel--hover" type="button">...</button>

Disabled button

Disabled buttons receive an .is-disabled class and a disabled attribute. 

<button class="button button--m button--flat bg-marvel is-disabled" disabled>Button</button>

Waiting button

Waiting buttons receive an .is-loading class. 

<button class="button button--m button--flat bg-marvel is-disabled" disabled>...</button>

Button with icon

Buttons with icons are wrapped in a parent component.uiIcon which has to have the button size applied to with an additional class. For example,.button--m will have a parent element with a class .uiIcon--m . The icon has to be placed straight after the button and should be an inline SVG.

Button 
<div class="uiIcon uiIcon--m">
    <span class="button button--m button--ghost bg-marvel--hover" role="button">...</span>
    <svg class="svgIcon">...</svg>
</div>

Button with lone icon

Unlike the button with icon, the button with lone icon doesn't have a parent componet but instead has the.uiIcon class added alongside.uiIcon--m and.button--m classes. 

<span class="uiIcon uiIcon--m button button--m button--ghost bg-marvel--hover" role="button">
    <svg class="svgIcon">...</svg>
</span>

Button with logo

Special case buttons used mainly for integrations with other services.

        <svg class="verticalAlign-middle">...</svg>
    </span>

    <span class="display-inlineBlock">...</span>
</button>

Stateful buttons

Stateful buttons have different states and behaviour to the regular ones. The pre-action feedback button, defined with.is-sad , will show a sad face on hover and is meant to be used on negative action buttons like deleting an account or downgrading. There are 2 post-click feedback buttons to be used for instant feedback and feedback after contacting a server. Those are defined by.is-stateful .

Hover me!

Pre-click feedback

Positive response

Negative response

<!-- Add class "is-sad" to activate the sad face on hover -->
<button class="button button--m button--ghost bg-negative--hover is-sad">...</button>

<!-- Add class "is-loading" on click until the server responds,
then "is-delayed", "has-succeeded" and "has-responded" -->
<button class="button button--m button--flat button--stateful bg-positive">
    <span>...</span>
</button>

<!-- Add class "is-loading" on click until the server responds,
then "is-delayed", "has-failed" and "has-responded" -->
<button class="button button--m button--flat button--stateful bg-marvel">
    <span>...</span>
</button>
